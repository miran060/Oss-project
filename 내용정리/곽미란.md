01강. 프로그래밍 기초 & 개발 환경
■ 프로그래밍 언어란?

컴퓨터와 소통하기 위한 표현 규칙

0과 1(기계어)을 사람이 이해하기 쉽게 추상화한 도구

목적: 문제 해결을 위한 명령을 컴퓨터에게 전달하는 방식

■ JDK / JRE / JVM 구분
구성 요소	설명
JDK	Java 개발 도구 (Java 컴파일러 포함) – 개발 시 필요
JRE	Java 실행 환경 – 자바 프로그램 실행에 필요한 라이브러리 제공
JVM	Java 바이트코드를 실행하는 가상 머신 – OS와 독립성 보장
■ 개발 환경 구성

JDK 설치 후 환경 변수 설정 (JAVA_HOME)

Eclipse/IntelliJ 등 IDE 설치

Java Project 생성 → 패키지 → 클래스 생성

■ Java 소스 구조

패키지 → 클래스 → main 메소드

main 메소드: 자바 프로그램의 시작점

주석 종류

// 한 줄

/*…*/ 여러 줄

/**…*/ 문서 주석

주요 키워드 예
public, class, int, if, while, static, new …

02강. 변수와 데이터 타입
■ 변수란?

메모리 공간에 붙여진 이름

지역 변수는 반드시 초기화 필요

■ 변수명 규칙

문자, 숫자, _, $ 사용 가능

숫자로 시작 불가

lowerCamelCase 사용

클래스 이름은 UpperCamelCase

예약어 사용 불가 (int, class 등)

■ 데이터 타입
✔ 기본 타입 (Primitive)
구분	타입
정수	byte, short, int, long
실수	float, double
논리	boolean
문자	char (유니코드 기반)
✔ 참조 타입

String

배열

클래스

인터페이스 등

■ 문자열(String)

문자열은 " "로 표현

주요 escape 문자
\n, \t, " , \

텍스트 블록: """ ... """ (자바 15+)

■ 타입 변환
✔ 자동 형변환

작은 타입 → 큰 타입 (범위가 넓은 타입)

byte → short → int → long → float → double

✔ 강제 형변환

(타입) 변수

✔ 주의

byte, short, char → 연산 시 자동으로 int 승격

■ 기본 입출력

입출력 클래스: Scanner

출력 메소드: println(), print(), printf()

03강. 연산자
■ 연산자 종류

부호/증감: +, -, ++, --

산술: +, -, *, /, %

비교: ==, !=, <, >, <=, >=

논리: &&, ||, !, ^

비트: &, |, ^, ~, <<, >>, >>>

대입/복합대입: =, +=, -=, *=, …

삼항 연산자: 조건 ? 값1 : 값2

■ 핵심 개념

오버플로우 / 언더플로우

정수/실수 비교 시 정밀도 문제 발생

문자열 비교는 ==가 아닌 equals()

5 / 0.0 → Infinity

5 % 0.0 → NaN

04강. 제어문 (조건문, 반복문)
■ 조건문
✔ if 계열

형태: if → else if → else

조건식의 true/false에 따라 분기

✔ switch 문

case 값에 따라 분기

break 없으면 fall-through 발생

문자/정수/String 가능

yield 사용 가능 (switch 표현식)

■ 반복문
✔ for

반복 횟수가 명확할 때

✔ while

조건이 true일 동안 반복

✔ do-while

최소 1번 실행 보장

■ 흐름 제어

break : 반복문 종료

continue : 다음 반복으로 건너뛰기

라벨 break : 중첩 반복문 한번에 종료

06강. 객체지향 프로그래밍(OOP)
■ 절차지향 vs 객체지향
절차지향	객체지향
“어떻게” 중심	“무엇” 중심
함수 중심	객체 중심
데이터보다 기능 우선	데이터와 기능을 묶음
■ OOP 핵심 4요소

캡슐화: 데이터 은닉, 접근 제한

상속: 코드 재사용

추상화: 공통 특징만 추출

다형성: 같은 메소드 호출 → 다른 동작 수행

■ 클래스 구성 요소

필드(Field): 객체 속성

생성자(Constructor): 객체 초기화

메소드(Method): 객체 동작

■ static

객체 없이 사용 가능

공용 데이터 저장

인스턴스 필드/this 사용 불가

■ final / 상수

final: 값 변경 불가

static final: 상수(대문자로 표기)

■ 패키지 / import

패키지: 클래스를 논리적으로 구분

import: 다른 패키지의 클래스 사용 시 필요

■ Getter / Setter

private 필드를 안전하게 접근·수정하기 위한 메소드

■ 싱글톤 패턴

프로그램에서 하나의 인스턴스만 유지

공용 객체 필요한 곳에 사용
